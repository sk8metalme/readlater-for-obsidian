📖 ReadLater for Obsidian - Slack通知機能テスト

🔍 テスト手順

ステップ1: 拡張機能のオプションページを開き、Slack Webhook URLを設定する
ステップ2: このページで右クリック → 後で読む（ReadLater） を選択
ステップ3: DevToolsのConsoleを開いてログを確認
ステップ4: Slackのチャンネルで通知を確認
⚠️ 注意:
  • Slack通知は個別保存モードでのみ送信されます
  • 集約保存モードでは通知は送信されません
  • 通知が失敗しても記事の保存は正常に完了します

📝 テスト用記事コンテンツ

Slack通知機能のテスト記事

公開日: 2025年10月14日

カテゴリ: テスト / 開発

概要

これはReadLater for ObsidianのSlack通知機能をテストするための記事です。 この記事を保存すると、設定されたSlackチャンネルに通知が送信されます。

通知内容

Slackには以下の情報が通知されます：

  • 記事のタイトル
  • 記事のURL（クリック可能なリンク）
  • 保存日時

期待される動作

  1. 記事が正常に保存される
  2. Slackに通知メッセージが送信される
  3. 通知にはタイトルとURLが含まれる
  4. 通知が失敗しても記事保存は成功する

トラブルシューティング

通知が届かない場合：

  • Webhook URLが正しく設定されているか確認
  • 「Slack通知を有効にする」がチェックされているか確認
  • 個別保存モードになっているか確認（集約保存モードでは通知されません）
  • DevToolsのConsoleでエラーメッセージを確認

🔗 リンク

⚙️ 設定画面を開く 📚 Slack Webhooks ドキュメント

✅ 確認ポイント

DevToolsのConsoleで確認すべきログ:

ReadLater for Obsidian: Starting article save process
ReadLater for Obsidian: Processing extracted article
ReadLater for Obsidian: Sending Slack notification { title: "...", url: "..." }
ReadLater for Obsidian: Slack notification sent successfully

📊 テストチェックリスト

以下の項目を確認してください：

  • ☐ Slack Webhook URLが設定されている
  • ☐ 「Slack通知を有効にする」がチェックされている
  • ☐ 個別保存モードが選択されている（集約保存モードはOFF）
  • ☐ 記事が正常に保存される
  • ☐ DevToolsで通知送信のログが確認できる
  • ☐ Slackのチャンネルに通知が届く
  • ☐ 通知に記事タイトルとURLが含まれている
